# 多维表格任务记录（暂不实现）

> **状态：⏸️ 现在不做** | **触发时机：团队规模扩大（>8 人）或需要任务跟踪时**

## 背景

当前请假交接 Bot 的完整流程：
```
@机器人 请假 → AI 解析 → 发卡片到群 → 点 ✅ 表情认领 → 认领确认
```

**缺失环节**：请假信息和认领记录没有持久化存储。卡片消息会被新消息冲走，无法回溯"谁请了什么假、谁认领了什么任务"。

## 未来方案：飞书多维表格

用飞书多维表格（Base）做请假与交接的持久化记录，替代当前的"卡片即记录"模式。

### 数据结构设计

| 字段 | 类型 | 说明 |
|------|------|------|
| 请假人 | 人员 | 从消息发送者获取 |
| 请假时间 | 文本 | AI 解析（明天/3天/下周） |
| 请假原因 | 单选 | 病假/事假/年假/回家/出差 |
| 待交接内容 | 多行文本 | AI 提取 |
| 风险等级 | 单选 | P0/P1/P2/P3 |
| 认领人 | 人员 | 表情认领时写入 |
| 认领时间 | 日期 | 表情认领时写入 |
| 状态 | 单选 | 待认领/已认领/已完成 |
| 卡片消息ID | 文本 | 关联群消息 |

### 实现路径

1. **创建多维表格**：用 `lark-cli base +create` 创建一张表
2. **请假时写入**：AI 解析后，调用 `lark-cli base +record-create` 写入一条记录
3. **认领时更新**：表情认领事件触发，更新记录的"认领人/认领时间/状态"字段
4. **卡片附带表格链接**：交接卡片里加一个按钮，跳转到多维表格视图

### lark-cli 命令预览

```bash
# 创建多维表格
lark-cli base +create --name "请假交接记录" --as user --profile leave-bot

# 写入记录
lark-cli base +record-create \
  --app-token "bascnxxx" \
  --table-id "tblxxx" \
  --fields '{"请假人":"张三","请假时间":"明天","风险等级":"P1"}' \
  --as user --profile leave-bot

# 更新记录（认领时）
lark-cli base +record-update \
  --app-token "bascnxxx" \
  --table-id "tblxxx" \
  --record-id "recxxx" \
  --fields '{"认领人":"李四","状态":"已认领"}' \
  --as user --profile leave-bot
```

## 为什么现在不做

1. **8 人小组，卡片够用**：群消息量不大，卡片不会被冲走，回溯需求低
2. **增加复杂度**：要管理表格 schema、记录 ID 映射、权限配置
3. **需要用户授权**：多维表格操作需 `--as user`，服务器上还要做 `lark-cli auth login`
4. **边际价值低**：当前"卡片 + 表情认领"已闭环，多维表格是锦上添花

## 触发条件

满足以下任一条件时，再考虑实现：
- 团队规模超过 8 人，群消息量大，卡片容易被冲走
- 需要按月/按人统计请假与交接情况
- 需要给 Leader 提供请假看板/报表
- 有审计或合规需求，需要留存请假记录
