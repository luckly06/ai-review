## 换仓库操作步骤

### 1. 用 GitHub CLI 设置 Secrets

```bash
# 替换 NEW_REPO 为你的新仓库名，如 luckly06/new-project
REPO="luckly06/新仓库名"

# === 方案 A：两套 Secrets（推荐） ===

# GitHub通知Bot（notify-feishu.yml 用）
gh secret set FEISHU_GITHUB_BOT_APP_ID    --repo "$REPO" --body "cli_aab1c52bbdb85bd6"
gh secret set FEISHU_GITHUB_BOT_APP_SECRET --repo "$REPO" --body "vFVjxsqlEWLw6BFb58MJ6bCdAER1ff3h"
gh secret set FEISHU_GITHUB_BOT_CHAT_ID   --repo "$REPO" --body "oc_2392f68ca918e85bb2a7ea880eb16b57"

# ai-review 机器人（ai-review.yml 用）
gh secret set FEISHU_REVIEW_BOT_APP_ID    --repo "$REPO" --body "cli_aab1fae329b99bb4"
gh secret set FEISHU_REVIEW_BOT_APP_SECRET --repo "$REPO" --body "8bvqmt4TFrRO7SqqjqvvncrPoIzYNQci"
gh secret set FEISHU_REVIEW_BOT_CHAT_ID   --repo "$REPO" --body "oc_e98c804bce08a68b7b6b841545da4441"

# 请假交接Bot（服务器 PM2 管理，凭证硬编码在 /opt/leave-bot/config.js，无需 GitHub Secrets）
# 服务器：华为云 ECS hcss-ecs-9cda（124.71.209.36）
# 目录：/opt/leave-bot/
# 进程：pm2 restart leave-bot | pm2 logs leave-bot
# 注意：飞书只允许一个 event bus 连接同一 app，本地和服务器不能同时运行

# === 方案 B：共用一套（简单，通知和审查结果都发到同一群） ===
# gh secret set FEISHU_GITHUB_BOT_APP_ID    --repo "$REPO" --body "cli_aab1fae329b99bb4"
# gh secret set FEISHU_GITHUB_BOT_APP_SECRET --repo "$REPO" --body "8bvqmt4TFrRO7SqqjqvvncrPoIzYNQci"
# gh secret set FEISHU_GITHUB_BOT_CHAT_ID   --repo "$REPO" --body "oc_e98c804bce08a68b7b6b841545da4441"

# 验证
gh secret list --repo "$REPO"
```

### 2. 复制 workflow 文件

```bash
# 在当前仓库执行
cp .github/workflows/notify-feishu.yml ../新仓库路径/.github/workflows/
cp .github/workflows/ai-review.yml ../新仓库路径/.github/workflows/
cp .github/scripts/ai-review.js ../新仓库路径/.github/scripts/
cp -r scripts/leave-bot/ ../新仓库路径/scripts/leave-bot/
```

### 3. Push 到新仓库

```bash
cd ../新仓库路径
git add .github/workflows/notify-feishu.yml .github/workflows/ai-review.yml .github/scripts/ai-review.js
git commit -m "feat: add feishu notification and AI review workflows"
git push origin main
```

### 4. 验证

```bash
# 验证通知 workflow（创建 Issue 触发）
gh issue create --repo luckly06/新仓库名 --title "测试飞书通知" --body "验证"

# 验证 AI 审查 workflow（创建 PR 触发）
# 需要先配置 MINIMAX_API_KEY
gh secret set MINIMAX_API_KEY --repo "$REPO" --body "你的MiniMax API Key"
```

## 凭证来源

| 凭证 | 获取方式 |
|------|---------|
| App ID / App Secret | 飞书开放后台 → 应用详情 → 凭证与基础信息 |
| chat_id | `lark-cli im +chat-list --as bot --format json` |
| GitHub Secrets | `gh secret set <NAME> --repo <REPO> --body <VALUE>` |

---

## 注意

- **群与机器人的对应关系由你决定**：每个机器人发到哪个群取决于你设置的 `CHAT_ID` Secret，没有固定绑定。换仓库时先确认：GitHub 事件通知发哪个群？AI 审查结果发哪个群？然后设置对应的 `CHAT_ID`。
- **不要重建应用**：直接复用现有 App ID / App Secret
- 如果飞书 App Secret 泄露，去飞书后台重置后更新所有仓库的 Secret
- 方案 A 中 `ai-review.yml` 的环境变量名需同步改为 `FEISHU_REVIEW_BOT_*`
