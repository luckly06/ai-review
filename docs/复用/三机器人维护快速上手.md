## 二、职责边界

### 1. GitHub通知Bot（notify-feishu.yml）
- **职责**：监听 GitHub 仓库事件（push/issue/PR），推送到飞书群
- **触发**：仓库事件自动触发
- **发到哪**：group-github 群（`oc_2392f68ca918e85bb2a7ea880eb16b57`）
- **依赖**：飞书 Bot 凭证（Secrets）+ GitHub Token（自动）

### 2. ai-review（ai-review.yml + ai-review.js）
- **职责**：PR 触发 → 抓 diff → 调 MiniMax AI 审查 → 评论回 PR + 推送飞书
- **触发**：PR 打开/更新时
- **发到哪**：审查结果推送到 group-learn 群（`oc_e98c804bce08a68b7b6b841545da4441`）
- **依赖**：飞书 Bot 凭证 + **MiniMax API Key**（`MINIMAX_API_KEY` Secret）+ GitHub Token

### 3.group-请假交接Bot（scripts/leave-bot/）
- **职责**：监听群消息 → 解析请假 → 风险评级 → 发卡片 → 表情认领 → 创建任务 → @Leader
- **触发**：群内 @机器人 + ✅ 表情
- **发到哪**：group-learn 群（`oc_e98c804bce08a68b7b6b841545da4441`）
- **依赖**：飞书 Bot 凭证 + lark-cli（leave-bot profile）+ Leader open_id

## 三、问题分类：在哪改？

### A. 配置/凭证问题 → 查 [飞书机器人凭证模板.md](file:///f:/ai_agent/dev2/xiaomi/ai/test08/ai-review/docs/复用/飞书机器人凭证模板.md)，不用改代码

| 问题 | 解决方式 |
|------|---------|
| 机器人发错群 | 改对应 workflow 的 `CHAT_ID` Secret |
| Secret 没设/设错 | `gh secret set <NAME> --repo <REPO> --body <VALUE>` |
| App Secret 泄露 | 飞书后台重置 → 更新所有仓库 Secret |
| 机器人没权限（如 task:task:write） | 飞书开发者后台申请 scope → 发布新版本 |

## 四、快速上手命令

### 请假 Bot 运维（华为云服务器 PM2）
```bash
# SSH 登录服务器
ssh root@124.71.209.36

# 常用 PM2 命令

```

## 五、AI API 配置说明

**ai-review 机器人用 MiniMax AI 做代码审查，需要 API Key：**

- **API 提供方**：MiniMax（`api.minimaxi.com`）
- **模型**：`MiniMax-M3`
- **Secret 名**：`MINIMAX_API_KEY`
- **注入位置**：[.github/workflows/ai-review.yml](file:///f:/ai_agent/dev2/xiaomi/ai/test08/ai-review/.github/workflows/ai-review.yml) 第 28 行，作为环境变量传给 [ai-review.js](file:///f:/ai_agent/dev2/xiaomi/ai/test08/ai-review/.github/scripts/ai-review.js)
- **使用位置**：[ai-review.js](file:///f:/ai_agent/dev2/xiaomi/ai/test08/ai-review/.github/scripts/ai-review.js) `callMinimax` 函数，`Authorization: Bearer ${MINIMAX_API_KEY}`

**换仓库时必须设置：**
```powershell
gh secret set MINIMAX_API_KEY --repo <新仓库> --body "你的MiniMax API Key"
```

**请假交接 Bot 和 GitHub通知Bot 不需要 AI API**，只用飞书 Bot 凭证。
