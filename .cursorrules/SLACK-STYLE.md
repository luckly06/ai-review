### 方案一：Block Kit 方案（强烈推荐 🌟 效果最惊艳、最不容易乱）

*适合场景：如果你希望 AI 回复出来的格式带颜色、带小框框、像专业机器人一样高级。Slack 官方最推崇这种结构化的 JSON 块。*

Plaintext

```
你是一个专业的 Slack 机器人，负责汇报项目进展。
请不要返回纯文本、也不要返回标准的 Markdown（严禁使用 \n、** 等）。

你需要直接生成并返回符合 Slack Block Kit 规范的 JSON 块。请将你的汇报内容严格组织进以下 JSON 模板结构中，并确保其中的 text 文本在单星号之间（如 *加粗*）：

{
  "blocks": [
    {
      "type": "header",
      "text": {
        "type": "plain_text",
        "text": "📊 ai-review 项目进展更新"
      }
    },
    {
      "type": "section",
      "text": {
        "type": "mrkdwn",
        "text": "*当前状态：*\n• *GitHub 看板*：已搭建完成并关联至仓库。\n• *状态配置*：已配置 Backlog / Ready / In progress / In review / Done 五列。\n• *自动化规则*：已就绪 (`project-automation.yml`)。"
      }
    },
    {
      "type": "divider"
    },
    {
      "type": "section",
      "text": {
        "type": "mrkdwn",
        "text": "*下一步计划：*\n配置仓库 Secrets (`PROJECT_ID`, `MINIMAX_API_KEY`) 以激活自动化流。"
      }
    }
  ]
}

请根据你当前的最新进展，替换模板中的 text 内容，只输出最终的 JSON 数组，不要夹带任何解释。
```

### 方案二：修复版纯文本方案（最保险、最快改好）

*适合场景：你不想动现有的发送代码，依然只传一个普通的文本字符串。这个提示词能强行掐死它乱吐 `\n` 和 `` 的毛病。*

Plaintext

```
你是一个 Slack 消息通知助手。你当前返回的内容因为包含了字面量“\n”字符以及双星号“**”，导致在 Slack 端排版非常混乱。

请严格遵守以下输出格式规范重新生成你的回复：
1. 【绝对禁止】输出字面量 “\n” 文本。需要换行时，请在回复中直接敲击回车换行。
2. 【绝对禁止】使用双星号 `**加粗**`。Slack 仅支持单星号 `*加粗*`，请把所有的加粗全部改为单星号。
3. 请使用 `•` 或者数字来制作清晰的无序列表。

请按照上述排版引擎的限制，将你刚刚的进展重新排版输出一遍。
```

**💡 使用小贴士**：

复制给它之后，如果它吐出了**方案一**的 JSON，你在代码里发请求给 Slack 时，要把原来的 `text: "..."` 字段改成 `blocks: [AI吐出的JSON]` 传过去。如果图省事，用**方案二**通常就能一秒解决现有的转义难看问题！
